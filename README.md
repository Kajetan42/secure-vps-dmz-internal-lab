## IMPORTANT CHANGE - ROADMAP

Due to previous complications with including the Hytale game server in the project, I decided to exclude it from the project due to my lack of Java programming skills, and the early stage of the game, which meant there wasn't extensive and easily accessible documentation or video material guiding me through how to create the script I wanted. I didn't want to force myself to do this, risk the security of my architecture, or pretend I fully understood what was happening behind the scenes of the code and the files involved in executing the script's commands.

For this reason, I have updated the ROADMAP and established new future objectives: deploying a Web Server based on the same mechanism previously used for the game server (i.e., tmux and systemd) which would allow me to send prompt to ChatGPT via the API, as well as retrieval of specific rows and columns from specific database tables. The communication will occur between the DMZ VPS and the Internal VPS through a VPN connection.

## Project Goals

The goal of this project is to practice network infrastructure security concepts in a hands-on environment. This project presents a simplified server infrastructure architecture in the form of an educational laboratory, including:
- a dedicated DMZ for public services
- an internal server with a high level of trust
- controlled communication between zones
- basic monitoring and log collection mechanisms

## Architecture
The project consists of two VPS servers:

### VPS DMZ
A server exposed to public traffic, acting as a DMZ.

Technical Specifications:
- vCPU: 6
- RAM: 12 GB
- Disk: 100 GB NVMe SSD
- OS: Ubuntu 24.04 LTS


Features:
- web server
- basic network security
- limited administrative access
- security log source

Security:
- firewall (UFW)
- Fail2Ban
- SSH (target: keys only, currently password)
- system event monitoring


### Internal VPS
Internal server with restrictive access policy.

Technical Specifications:
- vCPU: 6
- RAM: 12 GB
- Disk: 100 GB NVMe SSD
- OS: Ubuntu 24.04 LTS

Features:
- backend services (DB, API)
- central monitoring point
- infrastructure security management

Security:
- firewall (UFW)
- Fail2Ban
- SSH with limited access
- SIEM / HIDS


## Segmentation and connectivity
- servers are connected via a secure VPN (site-to-site) tunnel
- communication between the DMZ and the internal zone is strictly limited to essential services
- default access policy: deny by default
- administrative traffic is separated from public traffic


## Administrative Access Model (SSH)
The target security model includes:
- passwordless authentication
- key-based access only
- source restriction
- access to the internal server limited to defined IP addresses

## Monitoring and Logging

Planned/Implemented Elements:
- centralization of logs
- monitoring of login attempts (SSH)
- analysis of events generated by the firewall and Fail2Ban
- deployment of free SIEM solution


The goal is to simulate core SOC tasks:
- event detection
- log analysis
- alert correlation
- incident response

## Technologies and Tools:
- Linux (Ubuntu Server)
- UFW
- Fail2Ban
- SSH
- VPN
- SIEM / HIDS

## WAŻNA ZMIANA - ROADMAP

Ze względu na poprzednie komplikacje związane z włączeniem w projekt, serwera gry Hytale, zdecydowałem się na wyłączenie go z tego projektu ze względu na brak moich umiejętności z zakresu programowania Java, a także zbyt wczesnego etapu gry, a co za tym idzie braku obszernej i łatwo dostępnej dokumentacji oraz materiałów wideo kierujących jak stworzyć skrypt, który chciałem. Z tego powodu zdecydowałem się zaaktualizować ROADMAPĘ oraz ustanowić za przyszłe cele postawienie Web Servera, na takim samym mechaniźmie jak serwer do gry (tj. tmux, systemd), przez którego możliwe będzie wywoływanie promptów do ChataGPT, dzięki API, a także wyciąganie konkretnych wierszy i kolumn z określonych tablic baz danych. ## Cele projektu

Celem projektu jest przećwiczenie zagadnień związanych z bezpieczeństwem infrastruktury sieci w praktyce.
Projekt przedstawia uproszczoną architekturę infrastruktury serwerowej w formie laboratorium edukacyjnego, obejmującą:
- wydzieloną strefę DMZ dla usług publicznych
- serwer wewnętrzny o podwyższonym poziomie zaufania
- kontrolowaną komunikację pomiędzy strefami
- podstawowe mechanizmy monitorowania i zbierania logów

## Architektura

Projekt składa się z dwóch serwerów VPS:

### VPS DMZ
Serwer wystawiony na ruch publiczny, pełniący rolę strefy DMZ.

Parametry techniczne:
- vCPU: 6
- RAM: 12 GB
- Dysk: 100 GB SSD NVMe
- OS: Ubuntu 24.04 LTS


Funkcje:
- serwer gry (Hytale)
- podstawowe zabezpieczenia sieciowe
- ograniczony dostęp administracyjny
- źródło logów bezpieczeństwa

Zabezpieczenia:
- firewall (UFW)
- Fail2Ban
- SSH (docelowo: tylko klucze)
- monitoring zdarzeń systemowych

### VPS Internal
Serwer wewnętrzny z restrykcyjną polityką dostępu.

Parametry techniczne:
- vCPU: 6
- RAM: 12 GB

- OS: Ubuntu 24.04 LTS

Funkcje:
- usługi backendowe (DB, API)

- SIEM / HIDS


## Segmentacja i łączność

- serwery połączone są bezpiecznym tunelem VPN (site-to-site)
- komunikacja pomiędzy strefą DMZ a strefą wewnętrzną jest **ściśle ograniczona** do niezbędnych usług
- domyślna polityka dostępu: **deny by default**
- ruch administracyjny jest separowany od ruchu publicznego


## Model dostępu administracyjnego (SSH)

Docelowy model zakłada:
- brak uwierzytelniania hasłem
- dostęp oparty wyłącznie o klucze SSH
- ograniczenie źródeł dostępu
- dostępu do serwera wewnętrznego ograniczony do zdefiniowanych IP
